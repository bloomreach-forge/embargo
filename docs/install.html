How to install the embargo plugin

Prerequisites

These instruction assumes that you have an HST project based on the HST archetype, i.e. a Maven multiproject consisting of at least three submodules: cms, site and content.

Maven 2 repository

Add this repository to your root pom.xml (if it has not already been defined in a parent pom):

<repository>
  <id>hippo-forge</id>
  <name>Hippo Forge maven 2 repository.</name>
  <url>http://maven.onehippo.com/maven2-forge/</url>
  <snapshots>
    <enabled>false</enabled>
  </snapshots>
  <releases>
    <updatePolicy>never</updatePolicy>
  </releases>
  <layout>default</layout>
</repository>

Installation in CMS/Repository

  1. Add these two dependency to the pom.xml of your cms module:

    <dependency>
  <groupId>org.onehippo.forge.embargo</groupId>
  <artifactId>embargo-addon-frontend</artifactId>
  <version>...</version>
</dependency>

  2. Add these two dependency to the pom.xml of your cms module:

    <dependency>
  <groupId>org.onehippo.forge.embargo</groupId>
  <artifactId>embargo-addon-repository</artifactId>
  <version>...</version>
</dependency>

  3. In case you're upgrading from version 1.01.05 it's advised to also include:

    <dependency>
  <groupId>org.onehippo.forge.embargo</groupId>
  <artifactId>embargo-addon-repository-updaters</artifactId>
  <version>...</version>
</dependency>

  4. Rebuild your project. After startup, you should have the embargo workflow and an example embargo-enabled usergroup.

HST additional

  • In your hst-config.properties add the following property to you file (7.7 & 7.8): 

    cms.preview.security.delegation.enabled = true

    On default this property is set on false and is needed for the HST to successfully merge sessions to enable preview mode.
